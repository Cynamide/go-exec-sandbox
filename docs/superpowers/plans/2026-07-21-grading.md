# Outcome And Process Grading Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the single default grader with configured outcome and process grading checks, filters, metrics, and per-check results.

**Architecture:** Add `internal/grading` for filter pipelines, checks, metrics, and grading results. Keep existing stdout/artifact comparison behavior as configured checks.

**Tech Stack:** Go, regex, JSON decoding, CSV/Markdown comparison helpers, table-driven tests.

## Global Constraints

- Existing exact/artifact behavior remains available.
- Tasks with both outcome and process grading disabled are invalid.
- Filter pipelines run before checks.
- Process checks require captured trace events.
